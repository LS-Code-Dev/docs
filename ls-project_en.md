# 🚀 LS-OS: Development Environment Project

> **Status:** `KNOWLEDGE ACQUISITION & RESEARCH` 📚
> **Target:** Gathering technical requirements for low-resource systems
> **Vision:** Create the ultimate environment for coding and server management

---

## 💡 Current Phase: Research & Intelligence

At the moment, the **LS-Code-Dev** team is gathering technical information and testing different kernel architectures. We’re not just assembling an OS; we’re studying how to optimize every single clock cycle for developer workflows.

We are analyzing:

* Package management in minimal environments
* Driver compatibility for legacy hardware (32/64 bit)
* Automation methods for rapid deployment

---

## 🏗️ Planned Versions (Future Releases)

The project includes different editions tailored to user needs:

### 1. LS-OS Lite (Terminal Only)

* **Focus:** Maximum speed
* **Interface:** CLI only (Command Line Interface)
* **Use:** Extreme C/C++ coding, file management, systems with RAM < 512MB

### 2. LS-OS Standard

* **Focus:** Daily productivity
* **Interface:** Tiling Window Manager
* **Use:** Web development, scripting, smooth multitasking

### 3. LS-OS Server Edition

* **Focus:** Stability and uptime
* **Interface:** Headless (managed via SSH)
* **Use:** Local hosting, databases, school/home network management

---

## 💾 Supported Architectures

To avoid leaving any hardware behind, research covers:

* **x86_64 (64-bit):** For modern machines and full performance
* **i386 (32-bit):** For reviving old laptops and industrial hardware
